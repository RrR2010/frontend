# TASK: Handle Forbidden UX Flows

---

## DESCRIPTION

- Summary (1 sentence): Add a consistent frontend experience for 403 responses and authorization failures.
- Steps (max 5):
  1. Define the user-facing behavior for forbidden requests and denied screens.
  2. Handle 403 responses from API calls in the shared client layer or relevant hooks.
  3. Show a clear access-denied state on protected pages when needed.
  4. Preserve existing session state when the user is forbidden instead of logging them out unnecessarily.
  5. Make the error experience consistent across admin and tenant screens.
- Implementation hints:
  - Forbidden is not the same as unauthenticated.
  - Keep the message concise and avoid leaking internal policy details.
