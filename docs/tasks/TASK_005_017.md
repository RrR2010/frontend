# TASK: Implement Role-Aware UI Behavior

---

## DESCRIPTION

- Summary (1 sentence): Adjust navigation, visible actions, and administrative screens based on platform and tenant roles.
- Steps (max 5):
  1. Identify components that should vary by platform role or tenant role.
  2. Hide or disable actions the current user should not access.
  3. Make tenant-scoped screens reflect the active tenant context clearly.
  4. Adjust sidebar or navigation items for platform versus tenant experiences.
  5. Keep the UI state derived from the auth context instead of ad hoc checks.
- Implementation hints:
  - This is about reducing confusion, not enforcing security.
  - Keep the UI conditional logic centralized when possible so it does not spread across many components.
