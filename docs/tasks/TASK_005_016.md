# TASK: Implement Frontend Route Awareness for Authorization

---

## DESCRIPTION

- Summary (1 sentence): Add client-side route awareness so protected views are only reachable when the frontend auth state allows it.
- Steps (max 5):
  1. Review layouts and pages that currently rely only on `isAuthenticated`.
  2. Add route-level awareness for public, protected, platform-only, and tenant-only areas.
  3. Use the auth context to decide whether a user may enter a route or needs to be redirected.
  4. Keep the client-side behavior consistent with backend authorization intent.
  5. Avoid duplicating backend policy logic in the frontend.
- Implementation hints:
  - This is UX support, not security enforcement.
  - The backend remains the source of truth; the frontend only reduces friction and avoids obvious dead ends.

## DEFINITION OF DONE (DoD)
