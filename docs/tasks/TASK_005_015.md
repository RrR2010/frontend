# TASK: Sync Frontend Auth Types and Route Contracts

---

## DESCRIPTION

- Summary (1 sentence): Align frontend auth types, route expectations, and API contracts with the new platform and tenant authorization model.
- Steps (max 5):
  1. Review the current auth types and API responses used by the frontend.
  2. Extend the shared types to represent platform roles, tenant roles, active scope, and available contexts.
  3. Update API contracts to reflect protected/public route expectations where needed.
  4. Make the current tenant context explicit in the client state.
  5. Remove or isolate stale type assumptions that no longer match the backend.
- Implementation hints:
  - Keep the types close to the backend naming so the contract is easy to compare.
  - The frontend should not invent authorization state; it should consume the backend contract.
